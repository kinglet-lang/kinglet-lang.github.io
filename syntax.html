syntax

A sketch of the language.

Braces, value types, monomorphized generics, => expression bodies. The pieces below differ most from everyday C++; everything else is closer to C++ with the rough edges filed down.

  • Concepts — concept Foo<T> { ... } constrains generic calls.
  • Pattern matching — tagged enum variants; expr match { ... } postfix syntax.
  • Nullables — T? types; ?: coalesce; postfix ? unwrap; try/catch on failure.
  • Uniform call — f(x) and x.f() desugar to the same thing when one fits.
  • Pipe — x |> f passes x as the first argument to f.
